# Local Agent Workbench 可执行文件

## 已完成的内容

现有 React 前端已经封装为 Electron 桌面应用，Python/FastAPI 后端通过 PyInstaller 一并打包。普通用户运行打包产物时，不需要另外安装 Python、Node.js，也不需要手动启动 Uvicorn。

桌面版的工作方式如下：

```text
桌面窗口（Electron）
        │
        ├─ 启动打包好的本地 Python 后端
        ├─ 启动只监听 127.0.0.1 的本地页面服务
        └─ 将页面的 /api 请求转发给本地后端
                     │
                     ├─ SQLite 任务数据
                     ├─ 本地 Git 工作区
                     ├─ 本地命令和 MCP Server
                     └─ 用户配置的模型接口
```

页面、后端、数据库和任务工作区都在本机运行。需要注意：如果配置的是远程模型接口，提问内容仍会发送到该模型服务商；“本地桌面版”不等于“模型也完全离线”。

## Windows 直接使用

当前已经生成并验证过的 Windows x64 便携版位于：

```text
frontend\release\Local Agent Workbench-0.1.0-win-x64.exe
```

直接双击该文件即可。它会自动启动内置后端并打开桌面窗口，不要再单独运行 `uvicorn`。

这是本地构建且没有购买代码签名证书的程序，Windows SmartScreen 可能显示“未知发布者”。确认文件来自自己的构建目录后，可以选择“更多信息”再选择“仍要运行”。不要对来源不明的同名程序这样操作。

第一次启动可能比之后慢，因为便携包需要释放运行文件。关闭窗口时，桌面程序会一并关闭内置后端。

## 运行数据保存位置

桌面版不会把数据库写进安装包，而是写到操作系统的用户数据目录：

| 系统 | 默认目录 |
|---|---|
| Windows | `%APPDATA%\Local Agent Workbench\` |
| Linux | `~/.config/Local Agent Workbench/` |
| macOS | `~/Library/Application Support/Local Agent Workbench/` |

其中：

- `data/workbench.db`：任务、对话和运行记录。
- `data/model-secrets.json`：模型接口密钥。
- `backend.log`：桌面后端启动日志。

这些文件可能包含敏感信息，不要上传到 GitHub。卸载或替换可执行文件不会自动删除这里的数据。

## 从源码构建

构建过程必须在目标系统本机执行：

- Windows 版本在 Windows 上构建。
- Linux 版本在 Linux 上构建。
- macOS 版本在 macOS 上构建。

原因是 PyInstaller 不能在 Windows 上直接生成 Linux/macOS 后端二进制文件。Electron 界面代码是同一套，但内置 Python 后端必须针对目标系统构建。

### Windows 构建

要求：

- Python 3.10 或更高版本。
- Node.js 18 或更高版本，包含 npm。
- Git。

在项目根目录执行：

```powershell
py -3.12 -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r backend\requirements-desktop.txt
npm --prefix frontend install
npm --prefix frontend run desktop:package
```

生成位置：

```text
frontend\release\Local Agent Workbench-版本号-win-x64.exe
```

### Linux 构建

先安装 Python、venv、Node.js、npm 和 Git。以 Debian/Ubuntu 为例：

```bash
sudo apt update
sudo apt install python3 python3-venv git
```

Node.js 建议使用 Node.js 官方支持的 18 或更高版本。然后在项目根目录执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements-desktop.txt
npm --prefix frontend install
npm --prefix frontend run desktop:package
```

生成的 AppImage 位于 `frontend/release/`。赋予执行权限后运行：

```bash
chmod +x "frontend/release/Local Agent Workbench-"*.AppImage
"frontend/release/Local Agent Workbench-"*.AppImage
```

某些 Linux 发行版运行 AppImage 还需要安装 FUSE 兼容包，例如 Ubuntu 上的 `libfuse2`。

### macOS 构建

安装 Python、Node.js 和 Git 后，在项目根目录执行：

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r backend/requirements-desktop.txt
npm --prefix frontend install
npm --prefix frontend run desktop:package
```

生成的 `.dmg` 位于 `frontend/release/`。当前构建没有 Apple 开发者签名和公证，因此其他 Mac 上首次运行时可能被 Gatekeeper 阻止；正式分发需要配置 Apple Developer 证书和 notarization。

## 开发和验证命令

只运行桌面开发版：

```powershell
npm --prefix frontend run desktop:dev
```

只构建内置后端：

```powershell
npm --prefix frontend run desktop:backend
```

验证内置后端能否启动、响应并正常退出：

```powershell
npm --prefix frontend run desktop:smoke-backend
```

生成未封装的桌面目录，适合快速调试：

```powershell
npm --prefix frontend run desktop:package:dir
```

完整构建命令 `desktop:package` 会依次执行：

1. 用 PyInstaller 生成当前系统的后端可执行文件。
2. 用 TypeScript 和 Vite 编译前端。
3. 用 electron-builder 生成当前系统的桌面包。

## 外部程序依赖

桌面包已经包含 Python 后端，但以下功能仍然依赖电脑上的外部程序：

- 管理 Git 仓库需要安装 Git。
- Docker 隔离执行模式需要安装并启动 Docker。
- 用户自行配置的 `stdio` MCP Server，需要对应命令已经安装在电脑上。
- 调用远程模型需要网络以及有效的 Base URL、模型名称和 API Key。

如果只查看页面，不使用对应功能，这些外部程序不是全部必需的。

## 跨平台注意事项

- Windows 路径如 `D:\project` 不能直接拿到 Linux/macOS 使用，需要重新选择对应系统上的项目目录。
- SQLite 数据库中的任务工作区保存的是绝对路径，换系统后建议重新建立任务。
- Windows 可执行文件不能在 Linux 直接运行，Linux AppImage 也不能在 Windows 直接运行。
- 当前默认构建的是 x64。要支持 ARM64，需要在 ARM64 机器上构建，或额外配置 electron-builder 的架构目标，并确保 Python 依赖提供对应平台包。
- 当前使用 Electron 默认图标。如需正式发布，应添加 Windows `.ico`、Linux `.png` 和 macOS `.icns` 图标。
- 正式对外分发前，应分别配置 Microsoft 和 Apple 的代码签名，减少系统安全提示。
